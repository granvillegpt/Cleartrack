Firebase Custom Email Domain Setup Guide

Configure a custom email domain (e.g., noreply@cleartrack.co.za) for your Firebase authentication emails

Overview

By default, Firebase sends emails from noreply@cleartrack-1f6c6.firebaseapp.com. You can configure a custom domain like noreply@cleartrack.co.za to make emails look more professional and improve deliverability.

Benefits:
  • Professional appearance (emails from your domain)
  • Better email deliverability
  • Improved brand recognition
  • Reduced chance of emails going to spam
1 Access SMTP Settings in Firebase Console
  1. Go to Firebase Console
  2. Select your ClearTrack project
  3. Navigate to Authentication → Templates
  4. Click on "SMTP settings" (usually at the bottom of the templates list)
2 Add Your Custom Domain
  1. In SMTP settings, look for "Custom domain" or "Authorized domains" section
  2. Click "Add domain" or "Customize domain"
  3. Enter your domain: cleartrack.co.za
  4. Click "Continue" or "Add"
Note: Firebase may require you to verify domain ownership first. If so, you'll need to add DNS records (see Step 3).
3 Add DNS Records (Domain Verification)

Firebase will provide you with DNS records to add to your domain. You'll need to add these records in your domain registrar or DNS provider (e.g., GoDaddy, Namecheap, Cloudflare, etc.).

Common DNS records Firebase requires:

Type Name Value Purpose
TXT @ or cleartrack.co.za firebase=your-project-id Domain verification
TXT @ or cleartrack.co.za v=spf1 include:_spf.firebase.com ~all SPF record for email authentication
TXT _dmarc v=DMARC1; p=none; DMARC policy (optional but recommended)
⚠️ Important:
  • Firebase will show you the exact DNS records you need to add
  • DNS changes can take 24-48 hours to propagate
  • Make sure you copy the records exactly as Firebase provides them
  • You may need to contact your domain registrar or IT team to add these records
4 Verify Domain Ownership
  1. After adding DNS records, go back to Firebase Console
  2. Click "Verify" or "Check verification"
  3. Wait for Firebase to verify the DNS records (this can take a few minutes to several hours)
  4. Once verified, you'll see a green checkmark or "Verified" status
Tip: You can use online DNS checker tools like dnschecker.org to verify your DNS records are propagating correctly.
5 Configure Email Address in Templates
  1. Go back to Authentication → Templates
  2. Click on "Password reset" template
  3. In the "From" field, you should now be able to select your custom domain
  4. Change it from noreply@cleartrack-1f6c6.firebaseapp.com to noreply@cleartrack.co.za
  5. You can also customize the "Sender name" (e.g., "ClearTrack" instead of "not provided")
  6. Click "Save"
✓ Example Configuration:
  • Sender name: ClearTrack
  • From: noreply@cleartrack.co.za
  • Reply to: support@cleartrack.co.za (optional)
Alternative: Use External SMTP Provider

If Firebase's custom domain setup is too complex, you can use an external SMTP provider (like SendGrid, Mailgun, or AWS SES) and configure it in Firebase:

  1. Sign up for an SMTP provider (e.g., SendGrid, Mailgun)
  2. Get your SMTP credentials (host, port, username, password)
  3. In Firebase Console → Authentication → Templates → SMTP settings
  4. Click "Use custom SMTP server"
  5. Enter your SMTP provider credentials
  6. Test the connection
  7. Save the settings
Popular SMTP Providers:
  • SendGrid: Free tier: 100 emails/day
  • Mailgun: Free tier: 5,000 emails/month
  • AWS SES: Pay-as-you-go, very affordable
  • Postmark: Great deliverability, paid plans
Troubleshooting

Domain verification failing?

  • Wait 24-48 hours for DNS propagation
  • Double-check DNS records are added correctly
  • Use DNS checker tools to verify records are live
  • Make sure there are no typos in the DNS records
  • Contact your domain registrar if you're unsure how to add DNS records

Emails still coming from Firebase domain?

  • Make sure domain verification is complete (green checkmark)
  • Check that you've updated the "From" field in each email template
  • Clear browser cache and refresh Firebase Console
  • Wait a few minutes after saving changes

Emails going to spam?

  • Make sure SPF and DKIM records are properly configured
  • Consider adding DMARC records
  • Warm up your domain by sending emails gradually
  • Use a reputable SMTP provider if using external SMTP
